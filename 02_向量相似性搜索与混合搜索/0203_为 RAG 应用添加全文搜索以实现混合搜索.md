在上一节中，我们了解了如何利用向量相似度搜索来实现一个检索增强生成（RAG）应用。虽然纯粹的向量相似度搜索能发挥很大作用，也是对普通全文搜索的重大改进，但在实际生产场景中，它往往不足以保证足够高的质量、准确性和性能。

在本节中，我们将探讨如何改进检索器以获得更好的效果。我们还会研究如何在 RAG 应用中添加全文检索功能，以实现混合检索。

### 1. 全文检索索引

全文检索是数据库中的一种文本检索方法，已经存在很长时间。它通过关键词在数据中匹配内容，而非在向量空间中依据相似度进行匹配。在全文检索中要找到匹配结果，检索词必须与数据中的某个单词完全一致。

要实现混合搜索，我们需要在数据库中添加一个全文搜索索引。大多数数据库都有某种类型的全文搜索索引，在本书中我们将使用 Neo4j 的全文搜索索引。

```
driver.execute_query("CREATE FULLTEXT INDEX PdfChunkFulltext FOR (c:Chunk)
ON EACH [c.text]")
```

在这里，我们在:Chunk节点的text属性上创建一个名为PdfChunkFulltext的全文索引。

---

### 2. 执行混合搜索

混合搜索的思路是先进行向量相似度搜索和全文搜索，再合并结果。为了能对比两种不同匹配的得分，我们需要对得分进行归一化处理，具体做法是将每个搜索的得分除以该搜索的最高得分。

```
hybrid_query = '''
CALL {
	// vector index
	CALL db.index.vector.queryNodes('pdf', $k, $question_embedding) YIELD node, score
	WITH collect({node:node, score:score}) AS nodes, max(score) AS max UNWIND nodes AS n
	// Normalize scores
	RETURN n.node AS node, (n.score / max) AS score
	UNION
	// keyword index
	CALL db.index.fulltext.queryNodes('ftPdfChunk', $question, {limit: $k}) YIELD node, score
	WITH collect({node:node, score:score}) AS nodes, max(score) AS max
	UNWIND nodes AS n
	// Normalize scores
	RETURN n.node AS node, (n.score / max) AS score
}
// deduplicate nodes
WITH node, max(score) AS score ORDER BY score DESC LIMIT $k
RETURN node, score
'''
```

我们编写一条联合 Cypher 查询，先执行向量相似度搜索，再执行全文搜索。随后对结果进行去重，并返回前 k 个结果。

```
similar_hybrid_records, _, _ = driver.execute_query(hybrid_query, question_embedding=question_embedding, question=question, k=4)
for record in similar_hybrid_records:
    print(record["node"]["text"])
    print(record["score"], record["node"]["index"])
    print("======")
```
